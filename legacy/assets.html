Assets, Inventory & Procurement

Cataloging hardware, software, licensing, and purchasing decisions to support Summit Care Medical Clinic’s secure, compliant, and organized infrastructure rollout.

Assets & Inventory Overview

After completing the research and development phase, the next step was to define how Summit Care’s hardware, software, licensing, and vendor purchases would be tracked and organized. We began by taking stock of the clinic’s existing environment and recording devices, model names, quantities, and supporting notes in a shared Google Sheets inventory document stored in Google Drive. That spreadsheet serves as an early single place of truth for planning, budgeting, procurement, and technical visibility as the environment moves from a smaller office setup toward a more organized enterprise-ready standard.

This page introduces both the inventory side of the project and the procurement direction behind the final design. It captures what assets need to be tracked, how ownership and visibility will be maintained, what platforms and licensing are being selected, and how Summit Care will keep a cleaner record of the equipment and services supporting the network. It also introduces the high-level hardware and software direction chosen for servers, desktops, laptops, VPN access, cloud services, switching, wireless, and virtualization.

Hardware & Device Tracking

Each asset is planned to receive a physical asset tag generated from a label machine and be recorded manually in the inventory spreadsheet. Rather than relying on RFID or automated scanning, the current plan is to document assets by hand so the technical team can keep practical visibility into the device name, role, location, assigned user, serial number, model, status, warranty or licensing notes, and any supporting remarks needed for future support and procurement visibility.

Office Laptops & Desktops

Fixed desktops and mobile laptops will be tracked by hostname, assigned user, department, model, serial number, software image, and lifecycle status so Summit Care can scale devices more consistently across clinical and administrative teams.

Network Infrastructure

Routers, switches, access points, firewalls, servers, and backup infrastructure will all be recorded with their role, placement, model, configuration ownership, serial details, and procurement source so the network side of the environment remains just as visible as the end-user device side.

Mobile & BYOD Devices

Any laptops, mobile devices, or bring-your-own devices with access to wireless, VPN, or internal services will be tracked at a higher level so Summit Care can maintain visibility into who is connecting, what access they have, and which users own those endpoints.

Software & Licensing

Alongside hardware, we also maintain an inventory of operating system licensing, Microsoft services, security tooling, remote support software, PDF/document tools, and any platform needed to operate or support the environment long term.

  • Windows Server 2022 (Chosen over newer release options because it is the more stable and comfortable fit for this deployment) – Domain services, DNS, DHCP, VPN support workloads, and core server infrastructure
  • Microsoft 365 / Entra P1 (Hybrid identity, SSO, email, productivity, and cloud access) – Replacing the clinic’s Google Workspace direction with Microsoft-aligned services and tighter integration with the planned on-prem environment
  • Adobe Acrobat, Remote Support & Utility Software (Endpoint standards and support tooling) – Includes document tools, remote administration software such as TeamViewer-class tools, and workstation setup utilities
  • Windows 11, Server & Vendor Licensing Research (Sourced and compared across vendors) – Licensing review focused on cost, legitimacy, support coverage, and long-term manageability

Licensing research is treated as part of procurement itself. We compared vendor sourcing options for Windows 11, Windows Server, Microsoft services, Adobe, remote support tools, and other required software to make sure Summit Care can stay compliant, budget-aware, and properly supported without relying on loosely tracked purchases.

Procurement Selections & Technical Direction

Once the research phase narrowed our options, procurement planning focused on which hardware and software stack would best support Summit Care’s real-world environment. Below is a high-level view of the direction selected for implementation.

Core Server Platform

We selected a Dell PowerEdge R740XD class server with approximately 64GB of RAM and 8TB of storage to support Windows Server roles, Linux-based services, virtualization, backups, remote access services, and future growth.

User Endpoints

Planned user hardware includes Dell OptiPlex 7020/7040 desktops for fixed workstations and Dell Inspiron laptops for mobility and support flexibility, helping maintain a more consistent Windows endpoint standard across the clinic.

Networking & Connectivity

The networking stack includes a Cisco ASA firewall, a Cisco router, Cisco CBS 1300 switching, Ubiquiti access points, a configured office printer, and supporting network-attached systems with static IP assignments where needed. The access point choice reflects compatibility with the switching environment and the practical benefit of continuing with spare Ubiquiti hardware already available to the project.

Redundant Remote Access

We are planning for two VPN paths: a Cisco firewall-based VPN and a Linux-based software VPN hosted on the Dell server platform, so Summit Care has higher availability, better redundancy, and more flexibility for remote access and administration.

Virtualization Direction

We selected Proxmox as the preferred hypervisor because it is free to deploy, Linux-based, and better aligned with the skill set already available in the technical team.

Microsoft Service Direction

Procurement planning also supports the shift from Google Workspace to Microsoft services, including Microsoft mail integration, Entra P1, single sign-on, hybrid cloud identity, and a more unified user and licensing experience.

Recovery Support Hardware

A Synology NAS and UPS are part of the supporting asset set so Summit Care can retain local backup copies, keep the server stable during power issues, and maintain better disaster recovery readiness.
