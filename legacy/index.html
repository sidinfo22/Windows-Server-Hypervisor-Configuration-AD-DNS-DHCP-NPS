Discover Networking Excellence

This document introduces Summit Care Medical Clinic, a mock healthcare organization used as the foundation for a full virtual blueprint that upgrades a small doctor’s office from a basic SoHo setup into an enterprise-grade network. In this project, we step in as the head of the IT department, working alongside clinic leadership and the head doctor to define naming conventions, IP schemes, inventory standards, procurement planning, documentation expectations, and the broader technical direction of the environment.

From gap analysis and research to provisioning, deployment, cloud integration, cybersecurity, compliance, and risk management, this report walks through the many responsibilities involved in building a reliable healthcare network. It also highlights how we support uptime, configuration uniformity, recovery readiness, and future scalability so Summit Care’s infrastructure can grow as one coordinated system instead of a series of disconnected fixes.

Full Stack Configuration

Full Stack Configuration

For Summit Care Medical Clinic, this section highlights the major configurations we put in place across systems, network infrastructure, and cloud identity. Rather than treating the environment like separate moving parts, we configured the stack as one coordinated platform so Summit Care can benefit from cleaner operations, stronger security, better user management, and more predictable day-to-day support.

Systems Configuration

We configured Windows Server services for Active Directory, DNS, DHCP, NPS, certificate services, and RADIUS-backed authentication so Summit Care has centralized identity, address management, policy enforcement, and secure enterprise wireless access.

Network Configuration

We configured Cisco routing and switching with structured VLANs, subnetting, router-on-a-stick, segmentation, trunking, and standardized switch policies so traffic stays organized, secure, and easier to troubleshoot across the clinic.

Cloud Configuration

We integrated Summit Care’s on-prem Active Directory with Microsoft Azure Entra ID using the sync agent to enable password hash sync, password writeback, SSPR, single sign-on, Microsoft 365 access, Teams provisioning, P1 licensing, and cloud-based staff management from one connected identity workflow.

Designed, Documented, and Scaled

Designed, Documented, and Scaled

Beyond configuration alone, we also approached Summit Care Medical Clinic’s environment with long-term structure in mind. We designed the architecture to support clear operational standards, documented the environment so future work could be carried out consistently, and built in room for growth so expansion would feel planned instead of reactive.

Designed for the Environment

We designed Summit Care’s network architecture around the real needs of a growing medical office, including segmentation, wireless access, hybrid identity, infrastructure standards, and a cleaner operational layout for both staff and IT support.

Documented for Consistency

We documented configurations, addressing plans, naming conventions, topology decisions, and deployment standards so the clinic would have a usable reference point for support, troubleshooting, onboarding, and future upgrades.

Scaled for Growth

When Summit Care added an additional office, our design and documentation made it far easier to extend the environment with consistency. Because the blueprint was already structured for growth, the clinic could scale into the new location with less friction and much better continuity.

Summit Care Infrastructure Priorities

Patient Data Protection

We designed Summit Care’s environment around protecting patient information, internal operations, and staff access with clearer boundaries, authenticated wireless access, role-based permissions, and policies that reflect how the clinic actually works day to day.

Configuration Standardization

We supported Summit Care with cleaner configuration uniformity, stronger change control, and better documentation so systems stay consistent, supportable, and far less likely to drift through accidental or unauthorized changes.

Uptime & Recovery Readiness

We planned Summit Care’s uptime around failover, monitoring, backups, and recovery readiness so staff can continue reaching critical systems with less disruption and better preparedness when something goes wrong.

Hybrid Cloud Identity

Azure Entra Integration

For Summit Care Medical Clinic, we extended the on-prem Windows environment into Microsoft Azure Entra ID so identity and access management could work efficiently across both local infrastructure and cloud services. Using the Entra sync agent, we connected on-prem Active Directory to the cloud to support password hash sync, password writeback, self-service password reset, single sign-on, Microsoft 365 access, user lifecycle management, and stronger day-to-day administration for staff accounts.

Directory Synchronization

We synchronized on-prem Active Directory with Azure Entra ID so Summit Care could maintain a connected identity model across local systems and cloud services without duplicating user administration.

Cloud Identity Benefits

Password writeback, password hash sync, SSPR, and single sign-on gave Summit Care a more efficient support workflow by enabling password resets, account management, and application access from the cloud while staying tied to the local directory.

Microsoft 365 & Teams

We provisioned Microsoft 365 services, Microsoft Teams accounts, and Entra P1 licensing so staff could collaborate more effectively while IT retained centralized control over access, cloud apps, and user provisioning.

Summit Care Secure Access Architecture

We built Summit Care’s secure access model around Windows Server, Active Directory, and Network Policy Server so the clinic could support enterprise wireless authentication, separate staff and guest access, stronger policy enforcement, and more accountable user access across the environment.

RADIUS & Enterprise Wi-Fi Authentication

We built a RADIUS-backed authentication workflow using Windows Server, Active Directory, and NPS so staff devices can join secure enterprise wireless with identity-based access instead of shared passwords.

Staff & Guest Access Policy Design

Access at Summit Care was shaped around segmented staff and guest wireless, captive portal disclosures, policy-driven permissions, and safer separation between internal clinic operations and public-facing connectivity.

VPN Access for Staff & IT Support

We also supported remote staff and IT administration with secure VPN access into the office private network using solutions such as Cisco Meraki VPNs and Linux-based software VPNs, improving efficiency, remote support flexibility, and controlled access to private resources.

Cybersecurity, Data Protection & HIPAA Compliance

Cybersecurity, Data Protection & HIPAA Compliance

For Summit Care Medical Clinic, our security work extended beyond access control into testing, data protection, and HIPAA-focused review. We assessed the network for weaknesses, reviewed how patient and office data would be protected across the environment, and evaluated software, configurations, and operational practices so the clinic could move toward a more secure and compliance-aware infrastructure.

Network Testing & Security Validation

We tested Summit Care’s network posture by reviewing segmentation, wireless security, access controls, exposed services, and configuration weaknesses so issues could be identified early and reduced before becoming operational or security risks.

Patient & Office Data Protection

We focused on securing office records and patient data through stronger access boundaries, safer authentication workflows, improved network separation, and tighter control over which users, devices, and systems could reach sensitive information.

HIPAA-Aware Software & Compliance Review

We reviewed Summit Care’s software stack, supporting configurations, and administrative practices with HIPAA compliance in mind so the clinic could rely on tools and workflows that better support privacy, accountability, regulated data handling, and healthcare operations.
