Research & Development

Documenting the technical research, platform comparisons, vendor analysis, and planning decisions that shaped Summit Care Medical Clinic’s enterprise network design.

Research Overview

Before finalizing Summit Care’s architecture, we used a research and development phase to compare hardware vendors, cloud ecosystems, hypervisors, software platforms, deployment methods, licensing paths, and long-term support requirements. This page captures the technical comparisons and decision-making process that helped us determine which tools, platforms, and infrastructure choices best matched the clinic’s operational needs, budget, scale, and staff skill set.

  • Cisco vs Ubiquiti comparison for switching, routing, compatibility, and usability
  • Google Workspace vs Microsoft cloud services comparison for identity, email, collaboration, and hybrid integration
  • Hypervisor and server planning across Proxmox, Hyper-V, VMware, storage, virtualization, and backup needs
  • Endpoint provisioning, VPN, licensing, remote support, and staff capability analysis for operational readiness

Network Hardware & Platform Comparisons

A major part of the research phase involved comparing which network platforms would best support Summit Care’s size, growth plans, and staff familiarity. We looked beyond specs alone and considered compatibility, long-term support, ease of management, training overhead, and how well each platform would fit into a healthcare environment with security and uptime requirements.

Cisco vs Ubiquiti

We compared Cisco and Ubiquiti for routing, switching, VLAN support, subnetting, firewall compatibility, controller-based management, and long-term reliability. Cisco stood out for enterprise networking depth and compatibility, while Ubiquiti remained attractive for simpler management and cost-conscious deployment in selected layers.

Cost, Scalability & Support

We evaluated not just purchase price, but licensing overhead, support expectations, scalability, replacement cost, and how easily the selected hardware could expand as Summit Care grows into additional offices, users, and network segments.

Staff Familiarity

Internal capability mattered in our selection process. We reviewed the existing skills of team members with CCNA-level networking knowledge, familiarity with Ubiquiti equipment, and prior exposure to Proxmox and Windows infrastructure to reduce unnecessary complexity in day-to-day support.

Cloud, Email & Identity Research

Summit Care also needed a clearer direction for cloud services, email, identity management, and collaboration. We reviewed how different ecosystems would impact user access, support overhead, pricing, hybrid synchronization, and the clinic’s long-term operational model.

Google Workspace vs Microsoft

  • Compared Google Workspace against Microsoft for mail, productivity, collaboration, and admin tooling
  • Reviewed licensing, migration complexity, security controls, and user experience for staff
  • Determined Microsoft aligned better with the planned Windows domain and hybrid identity model

Hybrid Cloud Integration

  • Evaluated how Microsoft cloud services would integrate with on-prem Active Directory
  • Reviewed single sign-on, cloud app access, user management, and sync behavior
  • Confirmed the Microsoft direction better supported Summit Care’s identity, mail, and remote collaboration goals

Hypervisors, Servers & Storage Planning

Another major research area focused on how Summit Care’s server environment should be built, virtualized, and sized. We compared hypervisors, storage expectations, and server role requirements to make sure the final environment could support domain services, web/database functions, backup operations, and future scaling.

Proxmox vs Hyper-V vs VMware

We compared Proxmox, Microsoft Hyper-V, and VMware for pricing, usability, virtualization features, backup integration, and how well each platform matched the technical comfort level of the support team.

Server Role Requirements

We mapped requirements for Active Directory, DNS, DHCP, web hosting, database services, backup targets, and storage expansion so the chosen server would not be undersized for production needs.

Storage & Backup Capacity

Storage planning included backup retention, VM growth, shared data, and expansion needs such as 8TB-class storage targets to support domain data, web/database workloads, and recovery planning.

Software, Licensing & Endpoint Provisioning

Research for this project also covered the practical tools needed to deploy and manage user devices. We compared provisioning software, imaging approaches, licensing sources, and standard endpoint software so Summit Care could build a consistent workstation strategy across desktops and laptops. That review also included whether selected software and cloud-connected services were a good fit for HIPAA-aware healthcare operations before they were approved for use in the environment.

  • Compared Windows 11 deployment, imaging, cloning, and PXE-based provisioning workflows
  • Reviewed Windows 11, Windows Server, Microsoft, and supporting software licensing paths across vendors
  • Assessed standard endpoint software such as Adobe Acrobat, remote support tools, and device setup utilities
Comparison Area What We Evaluated Why It Mattered
Provisioning Tools Clonezilla, Rufus, Windows Deployment Services, PXE boot Standardizing workstation setup and faster user rollouts
Licensing Windows 11, Windows Server, Microsoft services, vendor sourcing Budget control, support coverage, and compliance readiness
Endpoint Software Adobe Acrobat, remote support tools, desktop standards Consistent user experience and supportability

Remote Access, VPNs & Support Tools

Remote access requirements extended beyond users alone. We also compared the tools Summit Care would need for secure staff access, remote IT administration, and day-to-day support operations for distributed users and future growth.

VPN Platform Comparisons

We compared Cisco, Meraki, Linux-based VPNs, pfSense-style software VPN options, and other software-versus-hardware approaches to balance security, ease of management, user experience, and cost.

Remote Support Tools

For desktop support and administration, we compared tools such as TeamViewer and AnyDesk to determine which platforms offered the right mix of reliability, security, usability, and support efficiency for IT staff.

Endpoint Use Cases

We also reviewed how desktops, laptops, and remote staff devices would actually connect to Summit Care resources so the selected tools would remain practical for both office productivity and support workflows.

Staff Capability & Operational Readiness

A key part of the research process involved looking at the actual skills available on the support side. Technology decisions were weighed not only against features and cost, but also against whether the internal team could realistically manage, troubleshoot, and scale the environment over time.

Team Readiness Review

82% Practical Alignment with Current Team Skill Set

This reflects how strongly the chosen direction aligned with existing staff experience in Cisco networking, Ubiquiti hardware, Windows environments, and Proxmox-based virtualization, helping reduce training friction and improve long-term support confidence.

Conclusion & Next Steps

By comparing vendors, cloud platforms, licensing paths, provisioning tools, VPN solutions, hypervisors, server requirements, and staff readiness, this research phase established a clearer direction for Summit Care’s network transformation. Instead of choosing tools in isolation, we evaluated how each platform would affect cost, compatibility, usability, supportability, and long-term growth. With these findings in place, the next step is to move into Assets, Inventory & Procurement so we can decide what equipment, software, licensing, and vendors will be formally selected, tracked, organized, and purchased for implementation.
