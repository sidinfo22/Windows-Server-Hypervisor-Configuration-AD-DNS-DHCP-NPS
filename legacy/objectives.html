Company Objectives & Gap Analysis Overview

Identifying the architectural gaps in Summit Care Medical Clinic’s current environment so the network can be upgraded for stronger security, better efficiency, and enterprise-level performance.

COMPANY BACKGROUND

Situated in the heart of New York City, Summit Care Medical Clinic provides essential healthcare services to a growing patient base. As a HIPAA-regulated medical environment, the clinic handles large volumes of sensitive data including patient records, billing information, scheduling data, and internal operational documentation. At present, the environment still reflects a small-office setup centered around a home Verizon router, shared wireless access, manually configured desktops, and limited infrastructure standardization across systems.

Day-to-day operations require a reliable, secure, and scalable network architecture that can support endpoint desktops, future user provisioning, secure remote work for doctors and staff, protected remote meetings, and a more uniform digital experience across the office. Summit Care also needs stronger uptime planning, easier device deployment, better user standardization, and a clearer path to growth. These business and technical needs form the foundation for moving away from a basic small-office setup into an enterprise-grade network infrastructure with better consistency, visibility, and long-term supportability.

PROJECT OBJECTIVES & GOALS

  • Host an Internal Web & Database Platform: Move away from relying fully on third-party hosting by introducing an internal web and database server environment that gives Summit Care more control over availability, performance, and supporting business services.
  • Standardize Endpoint Deployment & User Provisioning: Create a more uniform desktop environment so office devices can be deployed, configured, joined to the domain, and scaled more easily as new users and workstations are introduced.
  • Enhanced Network Security: Replace the single shared wireless password with stronger RADIUS-backed authentication, segmented access, better policy enforcement, and improved protection for internal systems and staff connectivity.
  • Microsoft Platform Migration: Transition the clinic from Google Workspace services to Microsoft services so email, productivity, collaboration, and identity management can be aligned under a more unified environment.
  • Hybrid Identity & Cloud Synchronization: Integrate on-prem Active Directory with Microsoft cloud services to support single sign-on, identity synchronization, cloud-based management, and a more scalable hybrid infrastructure model.
  • Scalable Voice over IP (VoIP) Infrastructure: Introduce business-grade VoIP services to improve communication and support future office growth with better call handling and networked telephony.
  • Redundancy, Disaster Recovery & High Availability: Improve continuity planning through backup strategy, disaster recovery preparation, static-IP-ready connectivity, and network design choices that reduce downtime risk.
  • Segmentation, Compliance & Data Protection: Build the project around VLANs, subnetting, cybersecurity controls, HIPAA-aware planning, and stronger protection for office and patient data throughout the environment.

GAP ANALYSIS

To effectively transform this small-office setup into an enterprise-grade network, we need to identify the gaps between the current state of the network and the desired target infrastructure that aligns with HIPAA mandates, best practices, and Summit Care’s growth goals. Below is a focused overview of the most important areas that require improvement.

Current State

  • Home Verizon router serving as the primary gateway
  • Shared staff wireless password with limited access control
  • Google Drive and cloud tools used without unified hybrid identity
  • Office desktops configured manually with limited uniformity
  • No enterprise firewall, proxy, IDS, or IPS protections in place
  • Minimal disaster recovery planning and limited backup structure

Target State

  • Enterprise-grade firewall & router
  • RADIUS-based authentication, VLANs, and subnetting
  • Hybrid cloud synchronization with Microsoft services
  • Dedicated Windows domain services plus web, database, and storage roles
  • Advanced IDS/IPS, data protection, and compliance controls
  • Microsoft platform migration, VoIP, and structured disaster recovery

NEXT STEPS

With the business goals, network gaps, and architectural targets now defined, the next step is to move into Research & Development. That stage will focus on the validation work behind the project, including testing possible technologies, comparing deployment approaches, evaluating integration options, and documenting the planning decisions that support Summit Care’s enterprise network transformation. This keeps the project grounded in practical implementation instead of assumptions alone.
